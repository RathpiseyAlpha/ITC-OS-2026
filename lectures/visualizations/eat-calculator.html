How to read the EAT Graph

  • The blue line is EAT — the average time per memory access — as the TLB hit ratio α (x-axis) changes.
  • Ideal (lower dashed line) — every access hits the TLB → 1 memory access = ttlb + tmem. This is the diagram's green hit path (1 trip).
  • No TLB (upper dashed line) — every access misses → 2 memory accesses = ttlb + 2·tmem. This is the diagram's red miss path (2 trips).
  • EAT is the weighted average of those two by α — it always sits between them. As α → 100%, EAT slides down toward Ideal.
  • Drag α (or the tmem/ttlb sliders) and watch the point move. The bars above show the same three numbers for the current settings; run the Custom sequence tab in the diagram to see a real α emerge from actual hits and misses.

What each part of the diagram means

  • CPU — runs the program and issues a logical (virtual) address for every memory reference.
  • logical address ⟨p, d⟩ — split into page number p (which page) and offset d (how far into the page). Only p is translated; d passes straight through.
  • MMU — memory-management unit: the hardware that translates p → frame. It holds the TLB and the page-table base register.
  • TLB — a tiny, fast cache of recent ⟨page → frame⟩ translations (4 entries here). Searching it costs ≈ ttlb and touches no RAM.
  • page table — the full page → frame map, stored in RAM. Reading it is a real memory access (tmem); needed only on a miss.
  • physical address ⟨f, d⟩ — the real RAM address: frame f from translation + the unchanged offset d.
  • frames (RAM data) — physical memory in fixed-size frames; each row shows which page currently lives there.
  • Arrows: green = hit path, red = miss path, and the amber token carries the running +ns time cost.

The three simulation tabs

  • Single access — send one address through the hardware. Pick TLB hit (1 trip), Random, or TLB miss (2 trips), then step through it.
  • Miss → Hit — access the same page twice: the 1st misses and loads the TLB, the 2nd hits. Shows why repeated accesses (locality) make a TLB pay off.
  • Custom sequence — type your own reference string (pages 0–7). The TLB starts empty and warms up; the running hit ratio α and average access time appear — that average is the EAT.

Why does a TLB miss cost two trips to memory?

1 trip
TLB hit: the TLB already stores ⟨page → frame⟩, so the hardware gets the frame number without touching RAM. The only memory access is reading the actual data. → ttlb + tmem
2 trips
TLB miss: the frame number is unknown, so:
Trip 1 — read the page table to look up the frame. The page table lives in RAM, so this is a full memory access.
Trip 2 — only now, with the frame known, read the actual data at ⟨frame, offset⟩ — a second memory access.
→ ttlb + 2·tmem

The page table is data in memory just like everything else — there is no magic shortcut to it. That extra page-table read is the whole reason a miss is slower, and exactly the +tmem penalty the EAT formula charges for the (1−α) fraction of accesses that miss.
