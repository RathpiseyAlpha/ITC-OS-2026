Demand Paging & Virtual Memory — Swap In / Swap Out

A page is brought into RAM only when needed. Pages live on the disk backing store; the valid–invalid bit says whether a page is currently in a frame. Watch how virtual memory can be larger than RAM, and how several programs share the same frames.
🎯 Learning objectives — what you should be able to do (click to minimize)
  • Explain demand paging: a page is brought into memory only when it is first referenced.
  • Trace a page fault: locate the page in swap, bring it into a frame, update the page table.
  • Follow victim selection and swap-out when no free frame is available.
  • Track how valid / invalid bits change as pages move between memory and disk.
