Semaphores — Producer / Consumer (Bounded Buffer)

Week 8 · Chapter 8 — build the classic solution one problem at a time: FIFO buffer → precedence → counting → mutex
🎯 Learning objectives — what you should be able to do (click to minimize)
  • Explain why a FIFO bounded buffer is needed between a producer and a consumer that run at different speeds.
  • Use a semaphore initialised to 0 to enforce a precedence constraint — the consumer can never read an empty buffer.
  • Use counting semaphores (chars = items, space = free slots) so a full buffer puts the producer to sleep instead of destroying data.
  • Show the race condition two producers cause on the shared index in, and fix it with a binary semaphore lock = 1 used as a mutex.
